## BAB V PENUTUP

### 5.1 Kesimpulan

Pengembangan sistem informasi kegiatan berbasis web untuk Pemerintah Kota Semarang telah berhasil menjawab permasalahan yang diidentifikasi pada website agenda asli yang bersifat statis dan tidak memiliki sistem manajemen konten. Sistem yang dibangun menggunakan teknologi modern dengan React TypeScript untuk frontend dan Laravel 11 untuk backend telah menghadirkan solusi komprehensif untuk pengelolaan agenda kegiatan pemerintahan yang lebih efisien dan terstruktur.

Implementasi sistem ini berhasil memenuhi seluruh tujuan penelitian yang telah ditetapkan. Pertama, sistem menyediakan Content Management System (CMS) yang memungkinkan pengguna untuk menambah, mengedit, dan menghapus kegiatan secara mandiri tanpa memerlukan keahlian teknis programming atau akses ke server. Interface yang intuitif dengan form input yang lengkap mencakup field judul kegiatan, tanggal pelaksanaan, waktu mulai dan berakhir, lokasi, jenis kegiatan, tingkat visibilitas, deskripsi detail, peserta internal, kontak eksternal, serta media foto dan video telah mempermudah proses input data kegiatan.

Kedua, sistem role-based access control (RBAC) dengan dua level pengguna yaitu atasan dan bawahan telah diimplementasikan dengan baik. Atasan memiliki hak akses penuh untuk melihat seluruh kegiatan termasuk yang bersifat private, melakukan approval terhadap kegiatan public yang dibuat bawahan sebelum dipublikasikan, serta mengelola registrasi pengguna baru. Bawahan dapat membuat kegiatan dengan berbagai tingkat visibilitas (public, private, kantor), mengelola kegiatan yang mereka buat, dan melihat kegiatan sesuai dengan aturan visibilitas yang berlaku. Mekanisme approval workflow memastikan bahwa kegiatan public yang akan ditampilkan di homepage telah melalui verifikasi oleh atasan sehingga menjaga kualitas dan kredibilitas informasi yang dipublikasikan.

Ketiga, fitur-fitur advanced yang mendukung produktivitas pengguna telah terimplementasi dengan baik. Sistem kegiatan berulang (recurring events) dengan frekuensi harian, mingguan, bulanan, dan tahunan memungkinkan pengguna untuk membuat satu entry kegiatan yang kemudian otomatis diekspansi menjadi multiple occurrences sesuai pola pengulangan, menghemat waktu input data untuk kegiatan rutin. Dukungan untuk kegiatan multi-hari dengan field tanggal mulai dan tanggal berakhir yang terpisah memungkinkan representasi yang akurat untuk acara yang berlangsung lebih dari satu hari. Fitur upload media dengan batas 50MB per file dan dukungan format gambar (JPEG, PNG, WebP, HEIC) serta video (MP4, AVI, MOV) memberikan kemampuan untuk mendokumentasikan kegiatan secara visual dan multimedia.

Keempat, sistem filtering dan pencarian yang komprehensif membantu pengguna menemukan informasi kegiatan dengan cepat. Filter berdasarkan OPD (Organisasi Perangkat Daerah), jenis kegiatan, dan tingkat visibilitas dapat dikombinasikan untuk menyaring agenda sesuai kebutuhan spesifik. Tampilan kalender dengan view bulanan memberikan overview visual terhadap distribusi kegiatan sepanjang bulan, dengan color coding berdasarkan jenis kegiatan untuk memudahkan identifikasi. Integrasi dengan field orang terkait memungkinkan pengguna melihat kegiatan yang melibatkan mereka meskipun bukan pembuat kegiatan tersebut.

Kelima, arsitektur sistem yang terpisah antara frontend dan backend dengan komunikasi melalui RESTful API memberikan fleksibilitas untuk pengembangan di masa mendatang. Frontend yang dibangun dengan React TypeScript memanfaatkan component-based architecture yang modular dan reusable, memudahkan maintenance dan penambahan fitur baru. Backend Laravel dengan pattern MVC dan Eloquent ORM menyediakan fondasi yang solid untuk business logic dan database operations. Penggunaan SQLite untuk development memudahkan setup dan testing, sementara schema database yang dirancang dapat dengan mudah dimigrasikan ke MySQL atau PostgreSQL untuk production environment dengan skalabilitas yang lebih tinggi.

Keenam, aspek security dan data integrity telah diperhatikan dengan serius. Password pengguna di-hash menggunakan bcrypt dengan salt otomatis, memastikan kredensial tidak pernah tersimpan dalam bentuk plain text. Validasi input di level backend mencegah data invalid masuk ke database, sementara authorization checks pada setiap endpoint API memastikan pengguna hanya dapat mengakses dan memodifikasi data yang sesuai dengan hak akses mereka. CORS configuration yang eksplisit mengontrol origin mana saja yang dapat mengakses API, mencegah unauthorized access dari domain yang tidak dikenali.

Sistem yang dikembangkan telah diuji melalui berbagai skenario penggunaan mencakup operasi CRUD kegiatan, approval workflow, recurring events expansion, multi-user collaboration, dan media upload. Testing menunjukkan bahwa sistem berjalan stabil dengan response time yang acceptable untuk operasi-operasi umum. Database indexing yang diterapkan pada field-field yang frequently queried seperti tanggal, pembuat, OPD, dan visibility memastikan performa query tetap optimal meskipun jumlah data bertambah.

Dengan demikian, dapat disimpulkan bahwa sistem informasi kegiatan berbasis web yang telah dikembangkan berhasil mengatasi limitasi website agenda asli dan menyediakan platform yang modern, user-friendly, dan feature-rich untuk pengelolaan kegiatan pemerintahan Kota Semarang. Sistem ini tidak hanya menjawab rumusan masalah yang diidentifikasi di awal penelitian, tetapi juga memberikan nilai tambah melalui fitur-fitur advanced yang meningkatkan efisiensi dan kolaborasi antar unit organisasi perangkat daerah.

### 5.2 Saran

Berdasarkan hasil implementasi dan evaluasi sistem informasi kegiatan yang telah dikembangkan, terdapat beberapa saran untuk pengembangan dan penyempurnaan lebih lanjut. Dari sisi fungsionalitas, sistem dapat diperluas dengan implementasi notifikasi real-time menggunakan WebSocket atau Server-Sent Events untuk memberitahukan status approval kegiatan secara instant, integrasi email notification untuk reminder otomatis H-1 sebelum kegiatan dan summary weekly agenda, serta pengembangan mobile application dengan React Native atau Flutter untuk akses yang lebih convenient dengan dukungan push notification. Enhancement pada user interface dapat dilakukan dengan menambahkan calendar view mingguan dan harian, drag-and-drop functionality untuk reschedule kegiatan, export ke format iCalendar untuk sinkronisasi dengan Google Calendar atau Outlook, dashboard analytics dengan visualisasi statistik kegiatan per OPD, serta fitur collaboration seperti comment thread pada kegiatan dan attendance tracking menggunakan QR code scanning.

Dari aspek teknis dan performa, sangat disarankan untuk melakukan migrasi database dari SQLite ke MySQL atau PostgreSQL untuk production environment, implementasi caching layer menggunakan Redis untuk query yang frequently accessed, database partitioning berdasarkan periode waktu untuk optimasi query temporal, serta integrasi full-text search engine seperti Elasticsearch untuk pencarian yang lebih powerful. Security enhancement juga perlu diprioritaskan melalui implementasi two-factor authentication (2FA) terutama untuk akun atasan, rate limiting yang lebih sophisticated berdasarkan sensitivity endpoint, regular security audit dan penetration testing, serta API versioning untuk backward compatibility. Pengembangan modul reporting advanced dengan kemampuan generate laporan PDF atau Excel, approval chain multi-level untuk kegiatan dengan budget besar, dan audit trail yang komprehensif akan sangat membantu keperluan administrasi dan dokumentasi.

Untuk mendukung ekosistem digital pemerintahan yang terintegrasi, sistem perlu dikembangkan dengan integrasi ke sistem existing seperti sistem kepegawaian untuk auto-populate data pegawai, sistem anggaran untuk tracking budget allocation, sistem asset management untuk booking ruang meeting, serta Single Sign-On (SSO) dengan centralized authentication server pemerintah. Dari perspektif accessibility, sistem harus memastikan compliance dengan Web Content Accessibility Guidelines (WCAG) untuk pengguna dengan disabilities, multi-language support Bahasa Indonesia dan Inggris, serta optimasi responsive design untuk berbagai ukuran layar. Establishment of user feedback mechanism melalui survey berkala dan user testing sessions, analytics tracking untuk user behavior, serta regular training dan workshop akan memastikan continuous improvement dan adoption rate yang tinggi sehingga sistem dapat terus berevolusi menjadi platform yang robust, scalable, dan user-centric.
